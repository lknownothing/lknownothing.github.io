Java快速开始-创建Maven项目并整合Tomcat发布Maven项目

August 18, 2020
Java, 笔记
Intro 依赖管理：maven工程对jar包的管理过程。 项目的一键构建：指的是项目从编译、测试、运行、打包、安装，部署整个过程都交给maven进行管理，这个过程称为构建。pom.xml文件。到pom.xml文件下面，执行`mvn tomcat:run’ 一、Maven 传统Web工程crm, Jar包在项目中， Maven开发的crm项目，jar包不在项目中。jar包的坐标-> jar包仓库。代码可重用， 1.1 Maven安装 安装只需要到官网下载解压即可。将解压路径 添加到MAVEN_PATH的环境变量中，path中添加%MAVEN_HOME%\bin config文件下的Default:${user.home}/.m2/repository会在home下建立repository文件夹，作为本地仓库的位置。当我们启动maven工程的时候，会默认取系统盘找本地仓库。连网情况下，会连接网络取中央仓库（放置了所有开源的jar包）寻找。也可以配置局域网，在远程仓库（私服）下载。 修改<localRepository>/path/to/local/repo</localRepository>本地仓库的位置。 1.2 工程目录 核心代码，配置文件，测试代码，测试配置文件。Maven目录标准目录结构 |-src/main/java目录 核心代码部分。 |-src.main/resource 配置文件部分 |-src/test/java 测试代码部分 |-src/test/resource 测试配置文件 |-src/main/webapp 页面资源，js,css,img等等。 1.3 Maven中常用命令 mvn clean将target目录删掉。 mvn compile 将src/main中的代码进行了编译。在公司中需要删除别人编译的信息，在自己电脑编译。 mvn test除了classes文件还有一个test-class文件。 mvn package 把打完的包放在了target目录下边，打包成一个war包，因为在pom文件中指定为war包。 mvn install 不仅编译main，还编译test, 还打包。 1.4 Maven生命周期 默认生命周期defaultLifeCycle:clean(清理生命周期), compile,test, install, 发布deploy。clean作为一个单独的清理周期；后面的default生命周期中，比如执行mvn test时就自动执行了前面的compile。 1.5 Maven概念模型图 包含三个信息：1) 项目本身的信息，2) dependency的jar信息，3) 项目运行环境信息，比如jdk，tomcat信息。 dependency: 公司组织的名称groupid，项目名字artifact，版本号version 每个构建项目的命令都对应了maven底层的一个插件。 1.
