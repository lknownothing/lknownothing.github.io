Java快速开始-JDBC

August 18, 2020
Java, 笔记

JDBC

一般的JDBC架构由两层模式组成：

  • JDBC API: 提供了应用程序对JDBC管理器的连接
  • JDBC Driver API：JDBC管理器对驱动程序的连接

下面是结构图：driver=com.mysql.jdbc.Driver

创建JDBC应用程序

构建一个JDBC应用程序包括以下6个步骤

  1. 导入数据：导入还有需要进行数据库编程的JDBC类的包。import java.sql
  2. 注册JDBC驱动器
    1. 方法1：使用java的Class.forName()来动态加载驱动程序的类文件,Class.forname和直接写DriverManager.registerDriver(new Driver)两者的功能等同。
    2. 方法2: DriverManager.registerDriver() 注册一个驱动程序。Driver MyDriver=new oracle.jdbc.driver.OracleDriver();
  3. 数据库URL制定：指向想要连接的数据库
  4. 打开连接：代码调用DriverManager对象的getConnection(string url,string user,string password)来创建一个Connection对象。

JDBC Statement对象

获得数据库的连接，就可以和数据库进行交互，JDBC的Statement，CallableStatement和PreparedStatement接口定义的方法和属性，可以发送SQL命令到数据库，并接受数据。Connection对象的createStatement()方法创建一个Statementstmt=conn.createStatement可以使用下面三个方法执行SQL语句

  1. boolean execute(String SQL)
  2. int executeUpdate(String SQL)
  3. ResultSet executeQuery(String SQL) 返回ResultSet对象。希望获得结果集时使用这个。

结果集。一个ResultSet对象控制一个光标指向当前行的结果集。Result接口的方法可以细分为三类：

  1. 导航方法
  2. 获取方法
  3. 更新方法

JDBC数据类型

VARCHAR为例，会映射为java.lang.String，NUMERIC会映射为java.math.BigDecimal。

JDBC事物

如果你的 JDBC 连接是处于自动提交模式下，该模式为默认模式，那么每句 SQL 语句都是在其完成时提交到数据库。conn.setAutoCommit(false);将自动关闭提交模式。

提交和回滚。完成修改，想要提交修改时，conn.commit()；回滚数据到数据库的时候conn.rollback; 下面的例子说明了如何使用提交和回滚对象.在这种情况下，之前的 INSERT 语句不会成功，一切都将被回滚到最初状态。有一个语句报错，数据库就不会变动。此外可以使用SavePoint对象达支持设定还原点。

try{
   //Assume a valid connection object conn
   conn.setAutoCommit(false);
   Statement stmt = conn.createStatement();

   String SQL = "INSERT INTO Employees  " +
                "VALUES (106, 20, 'Rita', 'Tez')";
   stmt.executeUpdate(SQL);  
   //Submit a malformed SQL statement that breaks
   String SQL = "INSERTED IN Employees  " +
                "VALUES (107, 22, 'Sita', 'Singh')";
   stmt.executeUpdate(SQL);
   // If there is no error.
   conn.commit();
}catch(SQLException se){
   // If there is any error.
   conn.rollback();
}

JSBC的转移义语法

  1. d,t,ts关键字。{d 'yyyy-mm-dd'}。它们能帮助确定日期，时间和时间戳的文字。众所周知，没有两个数据库管理系统的时间和日期的表现方式是相同的。该转义语法告诉驱动程序以目标数据库的格式来呈现日期或时间。此外，{t 'hh:mm:ss'}；
  2. escape关键字。该关键字在LIKE子句中适用，来定义转义字符。WHERE symbol LIKE '\%' {escape '\'}";告诉LIKE字符中的字符看作实际值，而不是通配符。LIKE 'AB&_%' ESCAPE ‘&'把&看成是转移字符。而“——”看成是实际的值。
  3. fn关键字。{fn length('Hello World')}代表在数据库中使用标量函数function。
  4. call关键字。对于需要一个IN参数的存储过程，使用{call my_procedure(?)};，对于返回一个OUT参数的存储过程，使用{? = call my_procedure(?)};。
  5. oj关键字。out-joint表示外部连接。
String sql = "SELECT Employees 
              FROM {oj ThisTable RIGHT
              OUTER JOIN ThatTable on id = '100'}";
stmt.execute(sql);

JDBC流数据

数据流。PreparedStatement对象必须具备使用输入和输出流来提供参数数据的能力。这使得能够将整个文件存储到数据库列中，这样的数据库能存储大型数据，例如CLOB和BLOB数据类型。用于数据流的有以下几种方法

  • setAsciiStream():该方法用来提供较大的ASCII值。
  • setCharacterStream():用来提供UNICODE值。
  • serBinaryStream(): 用来提供较大的二进制值。

Appendix

理解Class.forName() 。这里，不用解释，读者也会明白，在使用 jdbc 方式连接数据库时，为什么要执行 Class.forName(‘驱动类名’) 方法了：将驱动类的 class 文件装载到内存中，并且形成一个描述此驱动类结构的 Class 类实例，并且初始化此驱动类，这样 jvm 就可以使用它了，这就是 Class.forName() 方法的含义。

Java异常中Exception e中的getMessage()和toString()以及e.printStackTrace()方法的区别。由此可以看出，e.toString()获取的信息包括异常类型和异常详细消息，而e.getMessage()只是获取了异常的详细消息字符串。e.printStackTrace();会打出详细异常,异常名称,出错位置,便于调试用一般一个异常至少几十行

// Import required packages
import java.sql.*;
import java.io.*;
import java.util.*;

public class JDBCExample {
   // JDBC driver name and database URL
   static final String JDBC_DRIVER = "com.mysql.jdbc.Driver";  
   static final String DB_URL = "jdbc:mysql://localhost/EMP";

   //  Database credentials
   static final String USER = "username";
   static final String PASS = "password";

   public static void main(String[] args) {
   Connection conn = null;
   PreparedStatement pstmt = null;
   Statement stmt = null;
   ResultSet rs = null;
   try{
      // Register JDBC driver
      Class.forName("com.mysql.jdbc.Driver");

      // Open a connection
      System.out.println("Connecting to database...");
      conn = DriverManager.getConnection(DB_URL,USER,PASS);

      //Create a Statement object and build table
      stmt = conn.createStatement();
      createXMLTable(stmt);

      //Open a FileInputStream
      File f = new File("XML_Data.xml");
      long fileLength = f.length();
      FileInputStream fis = new FileInputStream(f);

      //Create PreparedStatement and stream data
      String SQL = "INSERT INTO XML_Data VALUES (?,?)";
      pstmt = conn.prepareStatement(SQL);
      pstmt.setInt(1,100);
      pstmt.setAsciiStream(2,fis,(int)fileLength);
      pstmt.execute();

      //Close input stream
      fis.close();

      // Do a query to get the row
      SQL = "SELECT Data FROM XML_Data WHERE id=100";
      rs = stmt.executeQuery (SQL);
      // Get the first row
      if (rs.next ()){
         //Retrieve data from input stream
         InputStream xmlInputStream = rs.getAsciiStream (1);
         int c;
         ByteArrayOutputStream bos = new ByteArrayOutputStream();
         while (( c = xmlInputStream.read ()) != -1)
            bos.write(c);
         //Print results
         System.out.println(bos.toString());
      }
      // Clean-up environment
      rs.close();
      stmt.close();
      pstmt.close();
      conn.close();
   }catch(SQLException se){
      //Handle errors for JDBC
      se.printStackTrace();
   }catch(Exception e){
      //Handle errors for Class.forName
      e.printStackTrace();
   }finally{
      //finally block used to close resources
      try{
         if(stmt!=null)
            stmt.close();
      }catch(SQLException se2){
      }// nothing we can do
      try{
         if(pstmt!=null)
            pstmt.close();
      }catch(SQLException se2){
      }// nothing we can do
      try{
         if(conn!=null)
            conn.close();
      }catch(SQLException se){
         se.printStackTrace();
      }//end finally try
   }//end try
   System.out.println("Goodbye!");
}//end main

public static void createXMLTable(Statement stmt) 
   throws SQLException{
   System.out.println("Creating XML_Data table..." );
   //Create SQL Statement
   String streamingDataSql = "CREATE TABLE XML_Data " +
                             "(id INTEGER, Data LONG)";
   //Drop table first if it exists.
   try{
      stmt.executeUpdate("DROP TABLE XML_Data");
   }catch(SQLException se){
   }// do nothing
   //Build table.
   stmt.executeUpdate(streamingDataSql);
}//end createXMLTable
}//end JDBCExample
